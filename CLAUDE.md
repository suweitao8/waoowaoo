# 项目规范

## 语言设置

**Claude 必须使用中文进行所有交流。**

- 所有对话、解释、代码注释、提交信息、文档等均使用中文
- 代码本身（变量名、函数名等）可使用英文，符合编程惯例
- 技术术语可保留英文，但应附带中文解释

## 版本号管理

**版本号存储在 `package.json` 的 `version` 字段中。**

### 版本号格式

使用语义化版本号：`主版本号.次版本号.修订号`（如 `0.4.1`）

### Git 提交时自动递增版本号

每次执行 git 提交时，Claude **必须**自动递增版本号：

- **patch 版本（修订号）**：Bug 修复、小改动、文档更新 → `0.4.1` → `0.4.2`
- **minor 版本（次版本号）**：新功能、功能增强、较大的重构 → `0.4.1` → `0.5.0`
- **major 版本（主版本号）**：重大架构变更、不兼容的 API 修改 → `0.4.1` → `1.0.0`

### 递增规则

1. 提交前，先修改 `package.json` 中的 `version` 字段
2. 默认情况下递增 patch 版本
3. 如果涉及新功能或较大改动，递增 minor 版本
4. 将版本号变更包含在同一个提交中

### 示例

```bash
# 修改 package.json 版本号后提交
git add package.json
git commit -m "fix: 修复登录问题"
```

## 开发服务器管理

**服务器的启动、关闭和重启由 Claude 自动完成。**

### 端口规范

**开发服务器必须运行在端口 3000。**

- 不允许在其他端口启动服务器
- 如果端口 3000 被占用，说明已有服务器在运行，需要先关闭再重启
- 关闭旧服务器的方法：`powershell -Command "Stop-Process -Id <PID> -Force"`

### 重启流程

当需要重启服务器时（如 Prisma schema 更新、依赖变更等），Claude 会自动：
1. 检查端口 3000 是否被占用
2. 如果被占用，先关闭占用该端口的进程
3. 清除 `.next` 缓存目录
4. 在端口 3000 启动开发服务器

无需用户手动操作终端。

## 前端错误处理规范

**禁止使用 `alert()` 函数显示错误信息。**

所有前端代码的错误信息必须使用控制台打印：

```javascript
// 正确做法 - 使用控制台打印
console.error('错误信息:', error);
console.warn('警告信息:', warning);
console.log('普通日志:', info);

// 错误做法 - 禁止使用 alert
// alert('发生错误');  // ❌ 不要这样写
```

### 原因

1. `alert()` 会阻塞 UI 线程，影响用户体验
2. 控制台打印便于开发调试和问题追踪
3. 生产环境可通过日志收集系统统一处理错误

### 用户友好提示

如果需要向用户展示错误信息，应使用项目中的 UI 组件（如 Toast、Message 等），而非 `alert()`。
