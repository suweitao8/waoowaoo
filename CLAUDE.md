# 项目规范

## 开发服务器管理

**服务器的启动、关闭和重启由 Claude 自动完成。**

当需要重启服务器时（如 Prisma schema 更新、依赖变更等），Claude 会自动：
- 停止当前运行的开发服务器
- 执行必要的清理操作（如清除缓存）
- 重新启动开发服务器

无需用户手动操作终端。

## 前端错误处理规范

**禁止使用 `alert()` 函数显示错误信息。**

所有前端代码的错误信息必须使用控制台打印：

```javascript
// 正确做法 - 使用控制台打印
console.error('错误信息:', error);
console.warn('警告信息:', warning);
console.log('普通日志:', info);

// 错误做法 - 禁止使用 alert
// alert('发生错误');  // ❌ 不要这样写
```

### 原因

1. `alert()` 会阻塞 UI 线程，影响用户体验
2. 控制台打印便于开发调试和问题追踪
3. 生产环境可通过日志收集系统统一处理错误

### 用户友好提示

如果需要向用户展示错误信息，应使用项目中的 UI 组件（如 Toast、Message 等），而非 `alert()`。
