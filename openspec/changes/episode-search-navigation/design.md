## Context

剧本分段面板（ScriptViewScriptPanel）当前展示所有 clips（分段/章节），用户通过滚动浏览定位目标章节。面板顶部有标题和分段计数，章节以卡片形式展示，每个卡片显示序号（从 1 开始）。

**现有结构**:
- 顶部标题栏：显示 "剧本分解" 和分段总数
- 卡片列表：每个卡片包含序号标签、编辑/删除按钮、剧本内容
- 选中状态：通过 `selectedClipId` 控制高亮

## Goals / Non-Goals

**Goals:**
- 添加章节跳转输入框，支持输入数字快速定位
- 输入数字后按回车或点击跳转按钮，滚动到对应章节
- 超出范围时给出友好提示（不超过范围时不显示错误）

**Non-Goals:**
- 不支持章节名称搜索（仅支持数字跳转）
- 不修改现有分段卡片布局
- 不影响现有的选中、编辑、删除功能

## Decisions

### 1. 输入框位置

**决定**: 放置在顶部标题栏右侧，分段计数旁边。

**理由**:
- 与分段计数并列，语义上关联（显示总数，支持跳转）
- 不占用卡片区域空间
- 保持现有布局结构

**备选方案**:
- 浮动按钮 + 弹窗输入 → 增加交互复杂度，不符合快速跳转目标
- 卡片列表上方独立行 → 占用垂直空间

### 2. 交互方式

**决定**: 输入框 + 回车确认 + 超出范围提示。

**交互流程**:
1. 用户输入数字
2. 按回车键触发跳转
3. 验证范围（1 ≤ 输入值 ≤ 章节总数）
4. 有效：滚动到对应卡片并选中
5. 无效：显示错误提示（2秒后自动消失）

**理由**:
- 回车确认符合用户习惯
- 即时反馈，无需额外按钮
- 错误提示不阻断操作

### 3. 滚动实现

**决定**: 使用 `scrollIntoView({ behavior: 'smooth', block: 'center' })`。

**理由**:
- 原生 API，无需额外依赖
- `block: 'center'` 将目标卡片居中显示
- 平滑滚动提升体验

**备选方案**:
- 使用 scroll 库 → 过度工程化
- 即时跳转 → 用户体验差

### 4. 错误提示样式

**决定**: 使用输入框下方的文字提示，主题色为危险色（danger）。

**理由**:
- 与现有 UI 风格一致
- 不使用 alert 弹窗（符合 CLAUDE.md 规范）
- 自动消失减少用户操作

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 输入非数字字符 | 限制 input type="number" 或使用正则验证 |
| 用户快速连续输入 | 跳转是即时操作，无防抖需求 |
| 卡片高度不一致影响滚动定位 | 使用 block: 'center' 尽量居中 |
