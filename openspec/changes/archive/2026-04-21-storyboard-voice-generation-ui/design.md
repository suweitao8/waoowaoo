## Context

当前剧本生成界面（ScriptViewScriptPanel）展示四种内容类型：
- `action` - 动作/分镜设计描述
- `dialogue` - 角色对话
- `voiceover` - 旁白
- 场景描述 - 场景头信息和描述

用户需求：
1. 移除 `action` 类型的分镜设计内容，统一改为 `voiceover`（旁白）
2. 在右侧增加语音生成和预览按钮

现有基础设施：
- 已有完整的语音生成系统（VoxCPM、百炼等）
- 已有角色音色绑定功能（`SpeakerVoiceBindingDialog`）
- 已有语音预览组件（`VoicePreviewSection`）

## Goals / Non-Goals

**Goals:**
1. 在剧本内容右侧添加语音生成和预览按钮
2. 根据内容类型（旁白/对话）智能识别并调用对应音色
3. 移除 action 类型渲染，简化剧本展示

**Non-Goals:**
1. 语音生成 API 的具体实现（用户后续调试）
2. 音色选择 UI（复用现有角色音色绑定）
3. 批量语音生成功能

## Decisions

### 1. UI 布局方案

**决定**：在每行内容（对话/旁白）右侧添加语音按钮

**理由**：
- 用户可在查看具体内容时直接生成语音
- 保持与现有编辑功能一致的交互模式
- 避免需要额外的选中/批量操作

**替代方案**：
- 全局批量生成按钮 → 需要额外的选择机制，增加复杂度
- 右侧独立面板 → 占用过多空间，与资源面板冲突

### 2. 按钮放置位置

**决定**：在每个 `dialogue` 和 `voiceover` 条目右侧内联显示按钮

```
[角色名] [对话内容...] [🔊 生成] [▶ 预览]
[旁白]   [旁白内容...] [🔊 生成] [▶ 预览]
```

**理由**：
- 一目了然，操作直接
- 无需额外选择步骤

### 3. 音色识别逻辑

**决定**：
- `voiceover`：使用默认旁白音色（需在设置中配置，暂时使用占位逻辑）
- `dialogue`：根据 `character` 字段查找角色绑定的音色

**理由**：
- 旁白通常使用固定音色
- 对话音色与角色绑定，符合现有工作流

### 4. action 类型处理

**决定**：不再渲染 `action` 类型内容，但保留数据结构

**理由**：
- 后端可能仍生成 action 数据
- 保持向后兼容
- 后续可根据需要添加转换逻辑

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 角色未绑定音色时无法生成对话语音 | 显示提示信息，引导用户先绑定音色 |
| 语音生成 API 未实现 | 按钮点击时显示占位提示，待用户调试后启用 |
| 大量内联按钮可能影响阅读体验 | 使用图标按钮，hover 时显示文字提示 |
