## Context

分镜（Storyboard）展示界面使用 `PanelEditFormV2.tsx` 组件显示每个分镜的编辑表单。当前表单包含：
- 景别（shotType）
- 镜头运动（cameraMove）← 需要隐藏
- 原文（sourceText）← 包含对话/旁白，需要解析展示
- 场景描述（description）
- 视频提示词（videoPrompt）← 需要隐藏
- 场景（location）
- 角色（characters）

剧本生成时，部分镜头可能既没有对话也没有旁白（纯镜头），用户希望这些转为旁白描述。

## Goals / Non-Goals

**Goals:**
- 隐藏 `videoPrompt` 和 `cameraMove` 字段的编辑界面
- 每个分镜必须显示对话或旁白内容
- 剧本生成时，没有对话/旁白的纯镜头转为旁白描述
- 分镜与剧本项一一对应

**Non-Goals:**
- 不修改数据库 schema
- 不影响视频生成流程（videoPrompt 仍会被使用）
- 不修改 API 接口

## Decisions

### 1. 字段隐藏方式
**决策**: 直接在 `PanelEditFormV2.tsx` 中移除对应字段的 JSX 渲染

**理由**: 最简单直接，数据仍保留在后端，不影响视频生成逻辑

### 2. 对话展示位置
**决策**: 在原文（sourceText）区域展示对话/旁白内容，作为必显示区域

**理由**: 每个分镜都必须有对话或旁白，这是核心内容

### 3. 纯镜头转旁白
**决策**: 在剧本生成的 prompt 中增加规则：没有对话也没有旁白的镜头，必须生成旁白描述

**理由**: 
- 从源头解决问题，确保每个镜头都有旁白或对话
- 不需要后续处理逻辑

### 4. 分镜与剧本对应
**决策**: 确保现有的一对一映射逻辑正确工作

**理由**: 当前系统已经支持分镜与剧本项的对应，只需确保新规则不破坏这一映射

## Risks / Trade-offs

**风险**: 对话解析可能不准确
→ **缓解**: 直接展示 sourceText 内容，不进行复杂解析

**风险**: 剧本生成的 prompt 修改可能影响现有生成质量
→ **缓解**: 仅增加规则，不修改核心生成逻辑
