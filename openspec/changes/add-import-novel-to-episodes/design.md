## Context

当前系统支持在创建项目时导入小说文件，系统会自动拆分章节并创建剧集。然而，用户在项目运行过程中无法追加新的小说内容，只能通过手动创建剧集并复制粘贴内容的方式添加新内容，效率低下。

现有的导入逻辑：
1. `SmartImportWizard` 组件处理小说导入流程（选择来源 → 解析 → 确认）
2. `workspace/page.tsx` 中的 `splitNovelByChapters` 函数处理章节拆分
3. 导入时使用 `importStatus` 状态控制流程显示

约束：
- 复用现有的小说导入逻辑，避免重复开发
- 导入的小说应追加到现有剧集列表末尾
- 不影响现有的剧集创建和管理功能

## Goals / Non-Goals

**Goals:**
- 在剧集选择器中添加"导入小说"按钮，与"新建剧集"按钮并列
- 复用 SmartImportWizard 或提取其核心逻辑用于追加导入
- 导入的小说章节作为新剧集追加到现有列表末尾
- 提供导入进度反馈和完成提示
- 导入完成后可选择跳转到新剧集

**Non-Goals:**
- 不修改现有的 SmartImportWizard 组件（保持项目初始导入逻辑不变）
- 不支持导入时的剧集合并或插入中间位置
- 不修改现有的 API 端点

## Decisions

### 1. 导入按钮位置

**决策**: 在 EpisodeSelector 组件的下拉菜单底部，与"新建剧集"按钮并列放置"导入小说"按钮。

**理由**:
- 用户直觉：剧集管理区域是添加内容的合理位置
- 保持 UI 一致性：新建和导入都是添加剧集的操作
- 替代方案考虑：放在顶部工具栏会增加用户认知负担，且与剧集管理脱节

### 2. 导入流程实现方式

**决策**: 创建 `AppendNovelWizard` 组件，复用 SmartImportWizard 的核心逻辑但简化流程。

**理由**:
- SmartImportWizard 与项目初始导入强耦合（依赖 `importStatus` 状态）
- 追加导入流程更简单：上传 → 解析 → 确认 → 追加
- 代码复用：提取共享的文件处理和章节拆分逻辑到工具函数
- 替代方案考虑：
  - 直接修改 SmartImportWizard 增加模式参数 → 会使组件逻辑复杂化
  - 完全独立实现 → 代码重复，维护成本高

### 3. 追加逻辑

**决策**: 在前端计算新剧集的起始编号，调用现有剧集创建 API 逐个添加。

**理由**:
- 复用现有的 `/api/novel-promotion/{projectId}/episodes` API
- 无需修改后端逻辑
- 提供实时进度反馈
- 替代方案考虑：创建批量追加 API → 增加后端复杂度，收益有限

### 4. 状态管理

**决策**: 在 `page.tsx` 中管理追加导入的状态，使用 React Query 刷新剧集列表。

**理由**:
- 与现有的剧集管理逻辑一致
- 利用已有的 `queryClient.invalidateQueries` 刷新机制
- 状态局部化，不影响其他组件

## Risks / Trade-offs

**风险 1: 大文件导入时用户等待时间过长**
→ 缓解措施：显示进度条和当前处理状态，提供取消功能

**风险 2: 导入过程中用户切换页面导致中断**
→ 缓解措施：导入过程中显示确认提示，防止意外离开

**风险 3: 章节编号与现有剧集冲突**
→ 缓解措施：自动计算起始编号，从现有剧集数量+1开始编号

**权衡**: 复用现有逻辑 vs 独立实现
- 选择复用：减少代码重复，但需要处理组件耦合
- 通过提取工具函数方式平衡两者
